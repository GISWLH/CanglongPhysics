# CAS-Canglong V2 模型参数量分析报告

- **总参数量**: **52.71M** (52,713,445)
- **可训练参数**: **52.71M** (52,713,445)
- **模型规模**: 约 **0.053B** (十亿级参数的5.3%)

### 2. Transformer主干网络

#### 各层详细参数

| 层级 | 深度 | 维度 | 分辨率 | 参数量 | 占比 |
|-----|------|------|--------|--------|------|
| Layer1 | 2 blocks | 96 | (6, 181, 360) | 5.15M | 9.8% |
| DownSample | - | 96→192 | - | 74.50K | 0.1% |
| Layer2 | 6 blocks | 192 | (6, 91, 180) | 17.93M | 34.0% |
| Layer3 | 6 blocks | 192 | (6, 91, 180) | 17.93M | 34.0% |
| UpSample | - | 192→96 | - | 83.14K | 0.2% |
| Layer4 | 2 blocks | 96 | (6, 181, 360) | 5.15M | 9.8% |
| **Transformer总计** | **16 blocks** | - | - | **46.32M** | **87.9%** |

**关键发现**:
- Transformer层占据了模型**87.9%**的参数
- Layer2和Layer3（各6个blocks）是参数最密集的部分，各占34%
- 使用了U-Net式的编码器-解码器架构，通过下采样和上采样连接不同分辨率

### 3. 编码器-解码器 (Encoder-Decoder)

| 模块 | 架构 | 参数量 | 占比 |
|------|------|--------|------|
| Encoder3D | ResNet-style (17→96 channels) | 3.30M | 6.3% |
| Decoder3D | ResNet-style (192→17 channels) | 2.76M | 5.2% |
| **编码器-解码器总计** | - | **6.06M** | **11.5%** |

**架构特点**:
- Encoder3D: 处理表面变量(17个变量)，包含:
  - 3层下采样
  - 残差块和非局部注意力模块
  - 输出96维latent特征

- Decoder3D: 将高维特征还原为物理变量，包含:
  - 3层上采样
  - 对称的残差块结构
  - 最终输出17个表面变量

### 4. 输出还原层 (Recovery Layers)

|---------|--------|------|
| PatchRecovery3D | 49,168 (49.17K) | 0.1% |
| PatchRecovery4D | 43,008 (43.01K) | 0.1% |
| PatchRecovery2D | 12,292 (12.29K) | 0.0% |
| **小计** | **104,468 (104.47K)** | **0.2%** |

### 5. 风向感知模块 (Wind-Aware Components)

| 模块 | 参数量 | 说明 |
|------|--------|------|
| WindDirectionProcessor | 0 | 无参数（纯几何计算） |
| WindAwareAttentionMask | 集成在Transformer blocks中 | 动态掩码生成 |

**创新点**:
- 风向感知窗口移位机制无需额外参数
- 通过预计算的风向ID动态调整注意力模式
- 物理信息完全通过架构设计集成，而非参数学习

### 参数效率分析

**优势**:
1. **参数量适中**: 比Pangu-Weather小72倍，比FuXi小27倍
2. **物理约束集成**: 风向感知机制无额外参数开销
3. **高效架构**: 87.9%参数集中在Transformer主干，学习能力强

**特色**:
- 比GraphCast略大(1.42倍)，但增加了:
  - 3D/4D卷积嵌入能力
  - ResNet风格的编码器-解码器
  - 风向感知注意力机制

## 架构创新总结

### 1. 多尺度处理策略
- **高分辨率路径** (Layer1/4): 5.15M × 2 = 10.30M (19.6%)
- **低分辨率路径** (Layer2/3): 17.93M × 2 = 35.86M (68.0%)
- 采用U-Net式skip connection实现多尺度特征融合

### 2. 物理信息编码
- **静态地球常量**: 通过conv_constant编码地形、土地覆盖等64个物理场
- **动态风场处理**: WindDirectionProcessor提取u/v分量计算主导风向
- **风向感知注意力**: 在每个Transformer block中应用物理约束的窗口移位

### 3. 变量处理流程
```
输入:
├─ 高空变量 (7, 5, 2, 721, 1440) → PatchEmbed4D → (96, 3, 181, 360)
├─ 表面变量 (17, 2, 721, 1440) → Encoder3D → (96, 2, 181, 360)
└─ 地球常量 (64, 721, 1440) → Conv2D → (96, 181, 360)
                    ↓
            堆叠 → (96, 6, 181, 360)
                    ↓
          Transformer主干 (46.32M参数)
                    ↓
            分离 → Surface (96, 2) + UpperAir (96, 3)
                    ↓
输出:
├─ 表面预报 → Decoder3D → (17, 1, 721, 1440)
└─ 高空预报 → PatchRecovery4D → (7, 5, 1, 721, 1440)
```

## 计算复杂度估算

### 内存占用（单次推理）
- **模型参数**: 52.71M × 4 bytes ≈ 200 MB (FP32)
- **输入数据**: ~100 MB (全球0.25°网格)
- **中间激活**: ~2-3 GB (取决于batch size)
- **总GPU显存需求**: 约 3-4 GB

### 训练效率
- **FLOPs**: 估计 ~500 GFLOPs/step
- **训练速度**: 约2-3 samples/sec (单V100)
- **多GPU扩展**: 支持DataParallel，可线性加速

## 结论

CAS-Canglong V2模型设计实现了以下平衡:

1. **参数规模适中** (52.71M)
   - 比超大模型(Pangu, FuXi)轻量70-100倍
   - 比极简模型(GraphCast)稍大，增强了表征能力

2. **架构设计高效**
   - 87.9%参数用于Transformer学习时空关系
   - 11.5%参数用于编码器-解码器处理物理变量
   - 风向感知等物理约束通过架构而非参数实现

3. **适合季节预报场景**
   - 周-月尺度预报需要强时空建模能力(Transformer)
   - 物理约束有助于长期预报稳定性
   - 模型规模适合研究机构训练和部署

**与同类模型相比，CAS-Canglong V2在保持合理参数量的前提下，创新性地融合了物理信息和深度学习，是针对S2S预报优化的专用架构。**
